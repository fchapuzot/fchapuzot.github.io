Premium feature

This section refers to a premium feature.

If you are working with the premium cloudify 3 plugin, then you have a feature that allows you to add and execute patches and custom operations on a deployed application. It needs to have a patches server deploy somewhere, and to makes some configurations.

Post-Deployment server URL

You must provide the endpoint URL of your deployed patches server: postDeploymentRestURL.

SSL configurations

If your server is deployed with SSL security, and required the clients to authenticate themselves before him, you need to provide alien4cloud with a bunch of security informations when configuring the orchestrator:

  • CA certificate: The authority certificate to refers to trust the patches server;
  • Client key: The key to use to authenticate the client before the patches server;
  • Client certificate: The certificate to use to authenticate the client before the patches server.

You must provide them in form of string data values (means you have to open your .pem file with a text editor and copy the content).

Make sure that you are in multi-line edition mode, by clicking on the icon next the edition box.
